# calculate_metrics.py Module Documentation

## Overview

This module calculates and reports performance metrics for AI model predictions on a multiple-choice question answering dataset. It takes prediction results from various models, compares them against ground truth labels, and computes metrics like precision, recall, F1-score, and accuracy. The results are then saved to a final results file for analysis.

## Architecture

This module is part of the data management pipeline within the SafeVixAI platform, specifically within the chatbot service. It processes the output of AI models (likely LLMs) that generate answers to multiple-choice questions. It reads prediction files, filters the data, calculates the metrics, and saves the results. It integrates with the data stored in Supabase and uses the LocalHashEmbeddingFunction for data integrity checks.

## Error Handling

*   **Size Mismatch:** Checks if the lengths of `y_true` and `y_pred` are equal and prints an error message if they differ.
*   **Invalid Labels:** Raises a `ValueError` if no valid labels (1, 2, 3, or 4) are found in the `correct_option_number` column.
*   **NaN Handling:** Uses `np.nan_to_num` to handle potential NaN values in precision, recall, and F1-score calculations.
*   **File Existence:** Checks if the final results file exists and removes it before starting the calculations.
*   **Data Filtering:** Filters results based on the presence of questions in the original dataset and valid labels.
*   **Type Conversion:** Converts prediction and correct option columns to integers.
*   **Correct Predictions Mismatch:** Flags an error if the number of correct predictions calculated in two different ways do not match.
*   **Total Predictions Mismatch:** Flags an error if the total number of predictions calculated in two different ways do not match.
