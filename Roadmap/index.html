    Phase 1 - Foundation (Done)

    Goal: Project structure, documentation, Git setup.

    • Create monorepo structure (backend/, frontend/, docs/)
    • Write all documentation (Agent.md, PRD, Architecture, API, DB, etc.)
    • requirements.txt and package.json with pinned versions
    • .gitignore, README.md, SETUP.md
    • GitHub repo created and initial push done
    • CI/CD workflow stubs (.github/workflows/backend.yml, chatbot.yml, frontend.yml, etc.)

    Phase 2 - Backend Core

    Goal: All 4 API modules working with real data.

    • Set up Supabase project and enable PostGIS + pg_trgm
    • Run Alembic migrations (create all 6 tables)
    • Seed traffic violations data (seed_violations.py)
    • Seed emergency services for 25 cities from OSM (seed_emergency.py)
    • Emergency Locator API - ST_DWithin + Overpass fallback
    • Geocoding API - Nominatim reverse geocode
    • Challan Calculator API - DuckDB SQL + state overrides
    • Road Reporter API - submit issue, route to authority
    • Set up Upstash Redis and connect cache client
    • Write tests for all endpoints (1365/1365 passing)

    Key files: - backend/api/v1/emergency.py - backend/api/v1/challan.py - backend/api/v1/roadwatch.py - backend/services/overpass_service.py - backend/services/challan_service.py - backend/services/authority_router.py

    Phase 3 - AI Layer

    Goal: RAG chatbot working with 9-provider fallback + ChromaDB.

    • Download 3 PDFs (MV Act 1988, MV Amendment 2019, WHO Trauma)
    • Run build_vectorstore.py to index PDFs into ChromaDB
    • RAG chain with ChromaDB MMR retrieval
    • Groq llama3-70b-8192 integration
    • 9-provider fallback chain: Groq→Cerebras→Gemini→GitHub→NVIDIA→OpenRouter→Mistral→Together→Template
    • Intent detection system (9 intent labels)
    • Chat history in Redis per session
    • Chat API endpoint - POST /api/v1/chat/
    • Chat stream SSE endpoint - POST /api/v1/chat/stream
    • Safety checker (7-layer defense)
    • Conversation summarization
    • Multi-turn intent refinement
    • Test RAG accuracy on sample queries (892/892 tests passing, 95% coverage)

    Key files: - chatbot_service/providers/router.py - chatbot_service/agent/graph.py - chatbot_service/data/build_vectorstore.py - chatbot_service/agent/safety_checker.py - chatbot_service/agent/intent_detector.py

    Phase 3b - External Integrations (API Expansion)

    Goal: Integrate rich external data sources to power backend logic and fallback workflows.

    • Integrate Open-Meteo API (Free) for visibility and precipitation logic.
    • Integrate Photon & BigDataCloud (Free) for robust zero-key frontend geocoding.
    • Setup OSRM for driving navigation and ip-api for dynamic state detection.
    • Connect What3Words (Key) and OpenCage (Key) fallbacks for critical SOS geolocation.
    • Connect Open FDA medical API tools for the first-aid persona.
    • Create Healthsites.io manual seeder script for the PostGIS backend.

    Phase 4 - Frontend

    Goal: All 4 modules working as connected UI.

    • Next.js App Router setup with TypeScript
    • Tailwind CSS global styles and design tokens
    • Home page with 4 module cards
    • Emergency Locator - GPS + MapLibre GL map + hospital markers
    • SOS button - calls nearest hospital
    • AI Chat - message bubbles, intent badges, source citations
    • Challan Calculator - violation selector, state dropdown, fine display
    • Road Reporter - photo upload, GPS tag, category select, submit
    • First Aid page - static offline guide
    • Zustand global store (location, chat history, offline status)
    • SWR data fetching with error/loading states
    • 28 routes, 91 components across 13 subdirs
    • 572/572 tests passing, 0 lint warnings
    • GSAP animations on all pages

    Key files: - frontend/app/emergency/page.tsx - frontend/app/assistant/page.tsx - frontend/app/challan/page.tsx - frontend/app/report/page.tsx - frontend/components/maps/MapLibreCanvas.tsx - frontend/components/chat/ChatInterface.tsx - frontend/lib/store.ts

    Phase 5 - Offline PWA

    Goal: All core features work with no internet connection.

    • Service worker + Workbox configuration (production only: npm run build && npm start)
    • Precache app shell (HTML, JS, CSS, fonts)
    • Cache india-emergency.geojson at install time
    • DuckDB-Wasm + violations.csv for offline challan
    • WebLLM Phi-3 Mini (2.2GB on-demand download, browser inference)
    • IndexedDB offline queue for SOS + road reports (auto-flush on online event)
    • Background Sync API for auto-submit when online
    • Offline status indicator in UI
    • @huggingface/transformers for YOLOv8 pothole detection (15MB ONNX)
    • Offline data bundle via /api/v1/offline/bundle
    • Test all modules in Chrome DevTools offline mode

    Key files: - frontend/lib/offline-ai.ts - frontend/lib/duckdb-challan.ts - frontend/lib/offline-sos-queue.ts - frontend/public/offline-data/

    Phase 6 - Deploy and Polish

    Goal: Live demo URLs working, submission ready.

    • Deploy FastAPI to Render.com (render.yaml)
    • Set all backend env vars in Render dashboard
    • Deploy Next.js to Vercel
    • Set all frontend env vars in Vercel dashboard
    • End-to-end testing (45/55 E2E tests passing)
    • Standalone build: copy-public.js auto-copies assets (fixes nested dir bug)
    • AuthGuard E2E bypass via __E2E_SKIP_AUTH__ localStorage flag
    • GSAP error fallback (try-catch in usePageEntry)
    • All CI workflows passing (backend.yml, chatbot.yml, frontend.yml, e2e.yml)
    • Record demo video
    • Final submission

    Phase 7 - V2 Features (Post-MVP, Complete)

    Goal: Win the hackathon with wow-factor features.

    • Bystander Mode (/bystander) — witness accident assistance with first-aid guidance
    • Family Live Tracking (/track, lib/live-tracking.ts) — Supabase Realtime GPS sharing
    • Share/Receive Location (/share-receive, lib/deep-link.ts) — deep link location sharing
    • QR Emergency Card (/emergency-card/[userId]) — scannable emergency profile
    • MCP Server integration (backend/api/v1/mcp_server.py) — external agent tools
    • Waze-style Traffic Feed (backend/api/v1/waze_feed.py) — community hazard data
    • Crash Detection engine (lib/crash-detection.ts) — DeviceMotion + GPS speed drop
    • Offline SOS Queue (lib/offline-sos-queue.ts) — IndexedDB + online event auto-flush
    • Turn-by-turn Navigation (lib/navigation-launch.ts) — multi-app navigation launcher
    • What3Words + OpenCage tools (chatbot_service/tools/what3words.py, geocoding.py)
    • Open-Meteo weather integration (chatbot_service/tools/open_meteo.py)
    • Drug Info tool (chatbot_service/tools/drug_info.py) — Open FDA medical data
    • Rate limiting on expensive endpoints (slowapi)
    • SOS incident persistence to database
    • Supabase RLS policies
    • Full JWT auth on chat/report endpoints
    • Service-to-service auth (X-Internal-Api-Key)
    • ALLOWED_HOSTS middleware
    • Circuit breakers for external APIs
    • Prometheus metrics endpoint
    • Progressive Guest Auth
    • CSP tightened (no 'unsafe-eval' in production)
    • SWR data fetching layer (7 cached hooks)
